# 基本設計書 v1.0 — 会議室予約システム

> 合成サンプル（下流資料）。要件定義 v0.9 を受けて作成。
> 検証用に「上流へ反映すべき内容」と「設計内に留める実装詳細」を混在させている。

## 1. システム構成
- Web アプリケーション（Spring Boot）＋ リレーショナルDB（PostgreSQL）で構築する。
- 社内ネットワーク内のサーバに配置する。

## 2. 認証・認可
- 既存の社内シングルサインオン（SSO / Active Directory 連携）を用いて認証する。
- 認証は社員IDで行い、「一般社員」と「会議室管理者」の2ロールを設ける。
  会議室管理者はマスタ（会議室の追加・休止）を編集できる。

## 3. 機能設計
### 3.1 予約管理
- 予約・変更・取消を提供する。
- 同一会議室・同一時間帯の重複予約は排他制御で防止し、競合した場合はエラーを表示して再入力を促す。
- 予約は24時間いつでも可能とする（毎日 2:00–2:30 のバックアップ時間帯を除く）。

### 3.2 空き状況閲覧
- 日／週カレンダーで表示する。画面上部に会議室検索ボックス、右側に会議室一覧を配置する。

### 3.3 会議室情報
- 収容人数を表示する。写真表示は今回のリリース対象外とする。

### 3.4 通知
- 予約確定時、および会議開始30分前に、予約者へメールで通知する。

## 4. 非機能設計
### 4.1 性能
- 主要画面の応答時間は3秒以内、想定同時接続数は100とする。

### 4.2 可用性
- 24時間稼働とする（バックアップ時間帯および月次計画停止を除く）。

### 4.3 データ
- 予約データは3年間保持し、その後アーカイブする。

## 5. データ設計（物理）
- reservations テーブル（id, room_id, user_id, start_at, end_at, status, created_at …）を中心に構成する。
- 検索性能のため (room_id, start_at) に複合インデックスを付与する。

## 6. 移行・他チーム調整
- SSO 連携のため、認証基盤チームへの接続申請・調整が必要。リードタイムは約2週間を見込む。
